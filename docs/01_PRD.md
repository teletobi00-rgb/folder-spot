# PRD — Windows 듀얼 페인 파일탐색기 (코드명: Explorer)

> 작성일: 2026-06-12 · 상태: 구현 진행 중 (Phase 0 완료) · 스택: .NET 10 (LTS) + WPF (확정)

## 1. 비전

Windows 탐색기를 대체하는 **듀얼 페인 + 페인별 탭 + 즉시 검색** 파일 매니저.
Total Commander의 생산성 키보드 모델과 Everything의 즉시 검색을 하나의 모던한(Fluent) 앱으로 합친다.

- 타겟 사용자: 파일을 많이 다루는 파워 유저(개발자, 자료 정리가 많은 사무직). 1차 사용자는 개발자 본인.
- 핵심 차별점: ① 페인별 독립 탭(TC 모델) ② 글로벌 핫키 즉시 검색(자체 인덱스, 네트워크 드라이브 포함) ③ 반대 페인 미리보기 탭

## 2. 필수 요구사항 (사용자 스펙)

| # | 요구사항 | 인수 기준 요약 |
|---|----------|----------------|
| F1 | 기본 파일 탐색 동작 전부 | 탐색/복사/잘라내기/붙여넣기/삭제(휴지통)/이름변경/새 폴더, 우클릭 셸 메뉴, 탐색기 ↔ 앱 드래그앤드롭, 주소창·breadcrumb, 뒤로/앞으로/위로, 드라이브 사이드바 |
| F2 | 2분할 화면 | 단일/듀얼 토글, GridSplitter 비율 조절, 활성 페인 시각 표시 |
| F3 | **분할 화면마다** 탭 | 페인이 탭을 소유(TC 모델). 추가/닫기/재정렬/반대 페인으로 이동, 세션 복원 |
| F4 | 즐겨찾기 (파일 + 폴더 단위) | 사이드바 핀 섹션. 폴더=이동, 파일=실행. 드래그로 추가, 재정렬, 영속 |
| F5 | 분할 화면 간 파일 관리 인터페이스 | F5 복사 / F6 이동(반대 페인으로), Tab 페인 전환, Ctrl+U 스왑, Ctrl+←/→ 반대 페인에 열기, 페인 간 D&D |
| F6 | 글로벌 핫키 파일 검색 + 사전 DB + 백그라운드 갱신 | 기본 핫키 **Alt+Space**(아래 주의 참조), 중앙 팝업, as-you-type, Enter=실행 / Ctrl+Enter=페인에서 위치 표시. 인메모리 인덱스 + SQLite 스냅샷, USN Journal 실시간 갱신 |
| F7 | 검색에 네트워크 드라이브 포함 | 드라이브별 opt-in, 주기 재스캔(분/시간/일/안 함) + FSW best-effort |
| F8 | 확장자별 색상 등 커스터마이즈 | 순서 있는 와일드카드 규칙(first-match-wins) → 색, 편집 UI. 테마(다크/라이트/시스템), 폰트, 행 높이 |
| F9 | 미리보기 (듀얼 모드 시 반대 페인에 미리보기 탭) | Ctrl+Q 토글(TC Quick View 방식). 활성 페인 선택을 250ms 디바운스로 추적. 이미지/텍스트·코드/미디어/압축 내장 렌더러 + Office·PDF는 IPreviewHandler |
| F10 | 컬럼 표시/크기 조절 | 파일명/확장자/크기/날짜/속성 표시·숨김·순서·너비·정렬, 폴더별 기억 |

> **⚠ F6 핫키 주의:** `Win+Space`는 Windows가 입력 언어(IME) 전환용으로 시스템 예약하고 있어 앱이 안정적으로 가로챌 수 없음(RegisterHotKey 실패 또는 미동작). PowerToys Run·Flow Launcher가 모두 `Alt+Space`를 기본값으로 쓰는 이유. → **기본 Alt+Space + 완전 재바인딩 지원 + double-Ctrl 프리셋** 제공으로 설계.

## 3. 추가 추천 기능 (제안)

### P0 — 필수 요구사항과 함께 구현 (v0.1~v1.0)
| 기능 | 이유 |
|------|------|
| 파일 작업 큐 UI | 진행률·속도·ETA·일시정지·취소. 대용량 복사 시 필수 체감 품질 |
| 커스텀 충돌 다이얼로그 | 덮어쓰기/건너뛰기/둘 다 유지 + "모두 적용" — 셸 기본 다이얼로그보다 듀얼 페인 워크플로에 맞음 |
| Undo (Ctrl+Z) | 이동·이름변경 되돌리기, 휴지통 복원. 실수 방지 핵심 |
| 퀵 필터 (Ctrl+F) | 현재 폴더에서 타이핑 즉시 필터 — TC 사용 빈도 최상위 기능 |
| Space 퀵 프리뷰 팝업 | 미리보기 탭과 렌더러 공유. macOS QuickLook 체감 |
| 세션 복원 | 종료 시 탭/경로/레이아웃 저장 → 재시작 복원 |
| 트레이 상주 + 자동 시작 | 메인 창을 닫아도 검색 핫키가 살아있어야 함 |
| 작업 히스토리 로그 | 언제 무엇을 복사/삭제했는지 추적 |

### P1 — v1.x 고도화
| 기능 | 이유 |
|------|------|
| 일괄 이름 변경 | 패턴(`{name}{counter}{date}`)/정규식/번호 매기기 + 미리보기 표 |
| 압축 파일 지원 | zip/7z/rar을 폴더처럼 탐색, zip/7z 생성·해제 (SharpCompress) |
| 검색 문법 | `ext:pdf size:>10mb date:2026 path:src regex:...` |
| 폴더 크기 계산 | 온디맨드 백그라운드 계산 + 컬럼 표시 |
| 중복 파일 찾기 | 크기 1차 → 해시 2차 그룹핑 |
| 관리자 권한 상승 재시도 | 보호된 경로 작업 실패 시 UAC 재시도 |
| Long-path 지원 | 260자 초과 경로(`\\?\`) — 작업 엔진 설계 시점부터 반영 |
| 다국어 (ko/en) / 포터블 모드 / 설정 내보내기·가져오기 | 배포 품질 |

### P2 — 백로그 (post-v1.x)
폴더 비교/동기화(페인 diff) · 색상 태그 시스템 + 태그 검색 · 파일 내용 전문 검색 · OneDrive placeholder 인식 · 플러그인 API · 원격 인덱스 질의(ETP 유사)

## 4. Non-Goals (v1 범위 제외)

- macOS/Linux 지원 (WPF = Windows 전용, 의도된 선택)
- 클라우드 스토리지 동기화 클라이언트, FTP/SFTP 내장 클라이언트
- RAR **쓰기** (독점 포맷 — 읽기만 지원)
- 파일 내용 인덱싱(v1은 파일명 검색만)

## 5. 성공 지표 (성능 예산)

| 항목 | 목표 |
|------|------|
| 대용량 폴더 | 10만 항목 폴더 스크롤 잰크 없음 (UI 가상화) |
| 검색 응답 | 로컬 100만 파일 인덱스에서 as-you-type < 50ms |
| 인덱스 구축 | NTFS 100만 파일 ≈ 1분 내 (관리자 MFT 경로), 인메모리 ≈ 140B/파일 (100만 ≈ 140MB) |
| 콜드 스타트 | SQLite 스냅샷 복원으로 < 2s |
| 권한 없는 환경 | 관리자 거부 시에도 **전 기능 동작** (인덱싱 속도만 저하) |
